Cross-Validation Guide

Apply two frameworks from different categories to the same question.

Pairings:
- "What should I pursue?" → Ikigai + Regret Minimization
- "Why am I stuck?" → Immunity to Change + Theory of Constraints
- "Is this the right decision?" → Pre-Mortem + Dialectical Thinking
- "Am I spending time well?" → Eisenhower Matrix + Double-Loop Learning
- "What am I missing?" → Johari Window + OODA Loop

Protocol:
1. Apply Framework A. Write down its conclusion.
2. Apply Framework B independently. Don't let A bias B.
3. Compare:
   - Agreement → high confidence, proceed
   - Partial overlap → overlapping part is solid, divergent parts need thought
   - Disagreement → gold. Ask "What assumption does each make that the other doesn't?" The gap is where the real insight lives.

When NOT to cross-validate:
- One framework clearly fits and the situation is simple
- You're over-analyzing to avoid action (if 2+ frameworks applied, stop and act)
- The user needs emotional support, not analytical rigor
